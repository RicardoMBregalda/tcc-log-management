Sistema de Gerenciamento de Logs

API REST - Arquitetura Híbrida (MongoDB + Hyperledger Fabric)

Documentação da API

Swagger UI OpenAPI Spec (JSON) OpenAPI Spec (YAML) Health Check

Características

  • Write-Ahead Log (WAL) com garantia de 0% de perda de dados
  • Sincronização assíncrona com blockchain Hyperledger Fabric
  • Cache Redis otimizado com TTL inteligente
  • Merkle Tree para verificação de integridade
  • Connection pooling otimizado (10-100 conexões)
  • Auto-Batching a cada 30 segundos
  • Redução de 80% na latência com sincronização assíncrona

Endpoints Principais

GET /health
POST /logs
GET /logs
GET /logs/{log_id}
POST /merkle/batch
GET /merkle/batch/{batch_id}
POST /merkle/verify/{batch_id}
GET /stats

Exemplo de Uso

Criar um novo log:

curl -X POST http://localhost:5001/logs \ -H "Content-Type: application/json" \ -d '{ "source": "app-server", "level": "INFO", "message": "Application started successfully" }'

Listar logs:

curl http://localhost:5001/logs?source=app-server&limit=10

Verificar saúde do sistema:

curl http://localhost:5001/health

Configuração

A API pode ser configurada através de variáveis de ambiente:

API_PORT=5001 MONGO_HOST=localhost POSTGRES_HOST=localhost REDIS_HOST=localhost FABRIC_API_URL=http://localhost:3000 AUTO_BATCH_ENABLED=true AUTO_BATCH_SIZE=50 AUTO_BATCH_INTERVAL=30
